Справка по системе и UI-фреймворк

Система управления контентом «Tagra CMS», далее просто система, создана для реализации веб-ресурсов (корпоративных, визиток, промо) с адаптивным дизайном и высокой производительностью. Также система может служить базой для разработки более масштабных и сложных веб-проектов. В отличии от более «тяжеловесных» CMS (Joomla, WordPress и т.д), использует меньше программного кода, библиотек, не перегружена лишним и проста в администрировании.

Структура и шаблон

В системе используются адаптивные шаблоны для комфортной работы и просмотра сайта на разнообразных устройствах. Чтобы ваш контент также был адаптивен, воспользуйтесь услугами профессионального верстальщика. С целью упрощения адаптивной верстки, в системе предусмотрена 12-ти колоночная модульная сетка, речь о которой пойдет ниже — в «Верстальщику»

В системе предусмотрена возможность создавать: «страницы», «разделы» и «галереи».
Страница — контент, оформленный с помощью текстового редактора, HTML, CSS, JS (JQuery), выводимый в шаблон системы (/application/views/front/page_view.php)
Раздел — контент, оформленный с помощью текстового редактора, HTML, CSS, JS (JQuery), выводимый в шаблон системы (/application/views/front/section_view.php). После контента пользователя следует перечень входящих в данный раздел страниц, галерей и других разделов с кратким описанием и ссылкой. В каждом разделе может быть неограниченное число подразделов, но с целью улучшения структуры сайта не следует злоупотреблять этой возможностью
Галереи — контент, оформленный с помощью текстового редактора, HTML, CSS, JS (JQuery), выводимый в шаблон системы (/application/views/front/gallery_view.php). После контента пользователя следует сама галерея. Возможны следующие типы галерей:
  • фото-галерея из папки
  • фото-галерея с описанием изображений
  • видео-галерея «ссылки YouTube»
  • аудио-галерея
Главная страница — контент, оформленный с помощью текстового редактора, HTML, CSS, JS (JQuery), выводимый в шаблон системы. Главная страница редактируется в админке отдельно от остальных страниц и выводится в отдельный шаблон системы (/application/views/front/home_view.php).
Страница «контакты» — контент, оформленный с помощью текстового редактора, HTML, CSS, JS (JQuery), выводимый в шаблон системы. После контента пользователя следуют контакты и форма обратной связи. Страница «контакты» редактируется в админке отдельно от остальных страниц и выводится в отдельный шаблон системы (/application/views/front/contact_view.php).
Header (шапка сайта) — не редактируется из админки, а доступна как шаблон для всего сайта кроме административной его части (/application/views/front/blocks/header_view.php). Содержит метатеги и фрагменты JS, CSS. Здесь же выводится и меню сайта.
Footer (подвал сайта) — не редактируется из админки, а доступен как шаблон для всего сайта кроме административной его части (/application/views/front/blocks/footer_view.php). Содержит фрагменты JS, информацию о правах и копирайте. Может также содержать продублированное меню или ссылки на другие ресурсы, контактные данные.
Основной CSS-файл — действует для всего сайта кроме административной его части (/css/front/general.css).
CSS-файл редактора— действует для редактора контента в административной части (/css/back/redactor.css).
Страница 404 — (/application/views/404_view.php).
Страница 403 — (/403.html).

Верстальщику

Система оснащена UI-фрейворком в котором предусмотрен CSS файл для стилистического оформления системы, и облегчения адаптивной верстки. А так же небольшой javasript файл. Эти файлы система использует повсеместно. Все файлы доступны в папке /UI_fraimwork. Внимание! Не изменяйте и не удаляйте файлы UI-фреймворка. Если вы хотите изменить стилевое оформление или javasript сценарий, редактируйте соответствующие файлы в /css/front и /scripts/front директориях или обратитесь к разаботчику по email в конце страницы. Адаптивная модульная сетка

Чтобы посетители могли комфортно просматривать контент на вашем сайте на разных устройствах, будь то смартфон или домашний кинотеатр — верстайте его, используя модульную сетку. Сетка может иметь неограниченное количество строк (row), каждая из которых может содержать до 12-ти колонок (col). 12 колонок = 100% от ширины шаблона сайта (устанавливается в админке — меню/конфигурация), 3 колоноки = 25% и так далее. Сделав поначалу каркас из сетки и разместив в нем контент, вы будете уверены в том, что при разной ширине устройств контент будет оставаться в рамках вашего каркаса, и ничто никуда не «поплывет». При ширине устройства >=600px все колонки сетки трансформируются в строки. Верстать в текстовом редакторе админки каркас из сетки и размещать в нем контент лучше в представлении «исходный код».

Переход к элементу
Чтобы плавно «прокрутить» страницу к желаемому элементу используйте функцию scrll(). Параметром она принимает строку - id элемента к которому нужно «прокрутить» и срабатывает по событию. Например: чтобы перейти к заголовку «Верстальщику» на этой странице, используется ссылка с onclick="scrll('verstalscheku')", а сам заголовок имеет id="verstalscheku". При переходе по ссылке которая имеет вид http://www.вашсайт/страница#idэлемента после загрузки страница будет плавно «прокручена» к элементу с id="idэлемента".

Лимит ввода

Если вы хотите ограничить количество символов, вводимых в поле, и показать пользователю, что лимит ограничен и сколько еще можно ввести — функция lim() поможет вам в этом. В теге input или textarea разместите onkeyup="lim(this,75)". Из аргументов здесь: "this" — указатель на обрабатываемое поле, "75" — лимит символов в данном случае.

default input default textarea

</label>
default textarea

    <textarea class="width80" placeholder="начните вводить символы" onkeyup="lim(this,75)"></textarea>
</label>
