Справка по системе и интерфейсу пользователя

Система управления контентом Tagra CMS, далее просто система, создана для реализации мультиязычных веб-ресурсов (корпоративных, визиток, промо) с адаптивным дизайном и высокой производительностью. Также система может служить базой для разработки более масштабных и сложных веб-проектов. В отличии от более тяжеловесных CMS (Joomla, WordPress и т.д), использует меньше программного кода, библиотек, не перегружена лишним и проста в администрировании.

Используемые скрипты, плагины и библиотеки

Система написана мной на базе PHP-фреймворка CodeIgniter. Чтобы не придумывать велосипед для решения некоторых задач в системе, я использовал скрипты, плагины и библиотеки с открытым исходным кодом, написанные другими разработчиками. Они опубликованы для свободного использования и распространения.

Список источников:
  • jQuery
  • TinyMCE WYSIWYG editor
  • MoxieManager
  • Emmet for <textarea>  | Документация и синтаксис emmet
  • Font Awesome

Структура и шаблоны

В системе используются адаптивные шаблоны для комфортной работы и просмотра сайта на разнообразных устройствах. Чтобы ваш контент также был адаптивен, воспользуйтесь услугами профессионального верстальщика. Чтобы упростить адаптивную верстку, в системе предусмотрен UI (интерфейс пользователя), речь о котором пойдет ниже — в Верстальщику

Языки

В системе предусмотрена возможность создавать мультиязычные материалы: страницы, разделы и галереи. Языки — условные версии веб-ресурса в которых все материалы, меню, страницы Главная и Контакты связаны по языковому признаку (принадлежат конкретному языку) и могут быть связанны с материалами из других языковых версий.

Страницы

Контент, оформленный с помощью текстового редактора, HTML, CSS, JS, выводится в шаблон системы (/application/views/front/page_view.php).

Разделы

Контент, оформленный с помощью текстового редактора, HTML, CSS, JS, выводится в шаблон системы (/application/views/front/section_view.php). После контента пользователя следует перечень входящих в данный раздел страниц, галерей и других разделов с кратким описанием и ссылкой.

Галереи

Контент, оформленный с помощью текстового редактора, HTML, CSS, JS, выводится в шаблон системы (/application/views/front/gallery_view.php). После контента пользователя следует сама галерея. Исходные файлы галереи (JS, CSS и прочие) находятся в папке /scripts/libs/FVGallery.

Возможны следующие типы галерей:
  • фотогалерея из папки (все изображения выбранной папки будут отображены в галерее)
  • фотогалерея с описанием изображений (изображениям в галерее можно задавать заголовок, описание и порядковый номер)
  • видеогалерея ссылки YouTube (в галерее отображаются видео с YouTube)
  • аудиогалерея (плеер с выбранными аудио)

Страница Главная

Контент, оформленный с помощью текстового редактора, HTML, CSS, JS, выводится в шаблон системы (/application/views/front/home_view.php). Страница Главная доступна для редактирования но создается/удаляется системой автоматически при создании/удалении языка.

Страница Контакты

Контент, оформленный с помощью текстового редактора, HTML, CSS, JS, выводится в шаблон системы (/application/views/front/contact_view.php). Страница Контакты доступна для редактирования но создается/удаляется системой автоматически при создании/удалении языка.

Header

Не редактируется из админки, а доступен как шаблон для всего сайта кроме административной его части (/application/views/front/components/header_view.php). Содержит метатеги и фрагменты JS, CSS. Здесь же выводится и меню сайта по умолчанию.

Footer

Не редактируется из админки, а доступен как шаблон для всего сайта кроме административной его части (/application/views/front/components/footer_view.php). Содержит фрагменты JS, информацию о правах и копирайте. Может также содержать продублированное меню или ссылки на другие ресурсы, контактные данные.

Верстальщику

Система оснащена интерфейсом пользователя (UI) в котором предусмотрен css-файл /Tagra_UI/style.css для стилистического оформления системы, и облегчения адаптивной верстки (откройте эту страницу в мобильном/планшете или измените ширину окна браузера чтобы видеть как UI адаптирует содержимое). А так же небольшой javasript-файл /Tagra_UI/script.js. Эти файлы система использует повсеместно. Все файлы UI доступны в папке /Tagra_UI. Внимание! Не изменяйте и не удаляйте файлы в /Tagra_UI Если вы хотите изменить стилевое оформление или javasript сценарий, редактируйте соответствующие файлы в /css/front и /scripts/front директориях или обратитесь к разаботчику по email в конце страницы.

Обратите внимание, что в начале файла стилей UI /Tagra_UI/style.css заданы переменные, которые определяют в интерфейсе пользователя цветовую схему, шрифт и прочее. Вы можете переопределить их значения чтобы, например, легко изменить цветовую схему элементов форм. Для этого скопируйте секцию ":root{...}" с переменными которые вы хотите переопределить и вставьте в начало основного css-файла /css/front/general.css, задайте свои значения переменным.

Адаптивная модульная сетка

Чтобы посетители могли комфортно просматривать контент на вашем сайте на разных устройствах, будь то смартфон или домашний кинотеатр — верстайте его, используя модульную сетку. Сетка может иметь неограниченное количество строк (row), каждая из которых может содержать до 12-ти колонок (col). 12 колонок = 100% от ширины шаблона сайта (устанавливается в админке меню/конфигурация), 3 колоноки = 25% и так далее.

Сделайте сначала каркас из сетки и разместите в нем контент, вы будете уверены в том, что при разной ширине устройств контент будет оставаться в рамках вашего каркаса, и ничто никуда не поплывет. При ширине устройства >=600px все колонки сетки трансформируются в строки. Верстать в текстовом редакторе админки каркас из сетки и размещать в нем контент лучше в представлении исходный код. В этом верстальщику поможет emmet, включить его можно на странице Конфигурация в админке.

Всплывающая подсказка
Особенность этой всплывающей подсказки  
          Привет, Мир!
          Это пример всплывающей подсказки,
          которая появляется потому, что на элементе
          (триггере), над которым сейчас расположен курсор,
          есть событие с вызовом метода TUI.Hint(this)
          onmouseover="TUI.Hint(this)"
в том, что:
  • она может быть вызвана по любому доступному событию на элементе (триггере)
  • триггером может быть любой элемент
  • скрытие подсказки допускается тоже любым событием, а можно и несколькими
  • содержимое подсказки может содержать HTML
  • содержимое подсказки будет индексироваться поисковым роботом
  • простой и акуратный код

Метод TUI.Hint() запускается по событию, установленному на триггере и показывает\скрывает подсказку, которая должна быть завернута в строчном элементе (span, pre). Элемент с подсказкой должен быть расположен сразу после триггера. Этому элементу необходимо присвоить class="hint".

Аргументы метода:
  1. (обязательный) указатель на триггер (как правило — "this", но можно указать "#idэлемента")
  2. (не обязательный) событие для сокрытия подсказки. Это событие следует передавать как строку без префикса "on", например: TUI.Hint(this,'click'). В данном случае, после появления подсказки, она будет скрыта только после клика по ней или любого иного элемента на странице. Можно передать несколько событий через пробел. Если второй аргумент (событие) не передано в метод, подсказка скроется когда с триггера будет убран курсор. Список стандартных HTML событий

Переход к элементу
Чтобы плавно прокрутить страницу к желаемому элементу используйте метод TUI.GoTo('#element'). Она принимает id элемента к которому нужно прокрутить и срабатывает по событию. Например: чтобы перейти к заголовку Верстальщику на этой странице, используется ссылка с onclick="TUI.GoTo('#verstalscheku')", а сам заголовок имеет id="verstalscheku". При переходе по ссылке которая имеет вид http://www.вашсайт/страница#idэлемента после загрузки страница будет плавно прокручена к элементу с id="idэлемента".

Формы

В UI предусмотрено стилизированое отображение элементов формы в одну строку (inline) и по одному этементу в строке, который используется по умолчанию, кроме кнопок. Каждый элемент формы должен быть помещен в тег <label> с классом соответствующему имени тега этого элемента. Чтобы элементы формы разместились в одну строку, примените класс "inline" к их родительскому тегу <label>. Чтобы выделить группу или одну кнопку, поместите их в <div class="button">.

Поиск по списку

Просто добавьте класс "SelectSearch" к элементу <select>, а UI прикрепит к нему поле поиска и обеспечит его работу. Используйте стандартные атребуты "size", "multiple" и другие, чтобы управлять списком как обычно. Сам плагин поиска (SelectSearch) находится среди прочих в файле /Tagra_UI/script.js. У вас есть список к которому вы хотите добавить поиск динамически? — Нет проблем! Справка в вкладке SelectSearch API

SelectSearch API

Свойства плагина (конфигурация)

Свойство Описание
el Селектор элемента/тов которые должен обрабатывать плагин.
Тип: строка.
Значение по умолчанию: 'select.SelectSearch'.
Пример использования: TUI.SelectSearch({el:'#mySelect'}) — найдет <select id="mySelect"> и прикрепит к нему поле поиска.
placeholder Сообщение в поле поиска.
Тип: строка.
Значение по умолчанию: 'Поиск по списку'.
Пример использования: TUI.SelectSearch({placeholder:'Что ищем?'}) — найдет все <select class="SelectSearch"> (свойство "el" по умолчанию) и активирует/изменит конфигурацию.
noresult Сообщение об отсутствии результатов поиска.
Тип: строка.
Значение по умолчанию: 'Ничего не найдено'.
Пример использования: TUI.SelectSearch({noresult:'Не нашлось'}) — найдет все <select class="SelectSearch"> (свойство "el" по умолчанию) и активирует/изменит конфигурацию.

Методы плагина

Метод Описание
init Активирует плагин на элементе/тах <select>. Если плагину не передан ни один метод, init выполняется по умолчанию.
Пример использования: TUI.SelectSearch() — найдет все <select class="SelectSearch"> (свойство "el" по умолчанию) и прикрепит к ним поля поиска.
kill Деактивирует плагин и удаляет поле поиска у элемента/тов <select>.
Примеры использования:
  • TUI.SelectSearch('kill') — найдет все <select class="SelectSearch"> (свойство "el" по умолчанию) и удалит поля поиска.
  • TUI.SelectSearch('kill', '#mySelect') — найдет <select id="mySelect"> и удалит поле поиска.
Поскольку UI использует библиотеку jQuery, вы можете применять методы этой библиотеки в цепочке вызовов с плагином. Например: TUI.SelectSearch({el:'#mySelect'}).attr('disabled', true) — найдет список <select id="mySelect">, активирует на нем плагин, а затем заблокирует список.
Лимит ввода

Если вы хотите ограничить количество символов, вводимых в поле, и показать пользователю, что лимит ограничен и сколько еще можно ввести — метод TUI.Lim() поможет вам в этом. В теге input или textarea разместите onkeyup="TUI.Lim(this,75)". Из аргументов здесь: "this" — указатель на обрабатываемое поле, "75" — лимит символов в данном случае.

Default input Default textarea
Пример кода
Default input
<label class="input">
  <input type="text" placeholder="начните вводить символы" onkeyup="TUI.Lim(this,75)">
</label>
Default textarea
<label class="textarea">
  <textarea placeholder="начните вводить символы" onkeyup="TUI.Lim(this,75)"></textarea>
</label>
Меню

Используйте обычный маркированный список с классом "menu". Если пункт не должен быть ссылкой, его следует поместить в теге <span>.
